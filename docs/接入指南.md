# qem-simplifier 适配指南（新 API 标准 / ABI v7）

> 本文档面向引擎插件、离线流水线与工具链集成方。
>
> 目标：基于 **场景图标准 API** 完成稳定接入，并明确写回、回滚、统计与迁移策略。

---

## 1. 适配范围与结论

从 ABI v7 起，场景能力的标准接入面为：

- 数据：`QemSceneGraphView`
- 接口：`qem_scene_graph_*`

执行类接口（单网格或场景）会原地修改几何数据并在成功时回写计数字段；失败时计数字段保持调用前值，但数组可能出现中间写入。

---

## 2. 接入前检查（强制）

### 2.1 ABI 门禁

初始化阶段必须调用 `qem_get_abi_version()`，仅当返回 `7` 时启用本指南流程。

### 2.2 动态库与符号检查

建议在加载期校验以下关键符号：

- `qem_context_create` / `qem_context_destroy`
- `qem_scene_graph_compute_decisions`
- `qem_scene_graph_apply_decisions_ex`
- `qem_scene_graph_simplify_ex`

### 2.3 运行时日志与进度回调

- 全局日志：`register_log_callback`
- 上下文进度：`qem_context_set_progress_callback`

建议在开发环境默认开启回调，便于定位容量错误与输入不合法问题。

## 3. 数据结构标准（接入侧必须满足）

### 3.1 `QemMeshView` 布局

- 顶点步长：`3 + num_attributes`
- `vertices` 长度：`num_vertices * (3 + num_attributes)`
- `indices` 长度：`num_indices`，且 `num_indices % 3 == 0`
- `material_ids` 长度：`num_indices / 3`
- 若 `num_attributes > 0`，`attribute_weights` 长度必须为 `num_attributes`

### 3.2 `QemSceneGraphView` 组成

- `meshes`：可写 mesh 池（执行阶段原地写回）
- `nodes`：节点层级（`parent_index` + `local_matrix[16]`）
- `mesh_bindings`：`node_index -> mesh_index` 映射

### 3.3 场景图约束建议

- `parent_index` 要么为 `-1`（根），要么指向合法节点
- `mesh_bindings` 的 `node_index/mesh_index` 必须落在合法范围内
- 同一 mesh 可被多个节点绑定（实例化语义）
- 启用 `use_world_scale` 时，建议保证节点矩阵可用于稳定尺度估算

---

## 4. 标准调用模式

### 4.1 模式 A：分步（推荐生产）

适用于预算审计、策略干预、可回放流程：

1. `qem_scene_graph_extract_features`
2. `qem_scene_graph_compute_decisions`
3. （可选）业务侧修改 `QemSceneMeshDecision[]`
4. `qem_scene_graph_apply_decisions_ex`
5. `qem_scene_compute_statistics`
6. `qem_scene_export_statistics_csv`

### 4.2 模式 B：一步（快速接入）

适用于工具化或快速验证：

1. `qem_scene_graph_simplify_ex`
2. （可选）`qem_scene_compute_statistics`

### 4.3 单网格模式

1. `qem_context_create`
2. 配置 `QemMeshView` + `QemSimplifyOptions`
3. `qem_simplify`
4. 成功后读回前缀数据并持久化
5. `qem_context_destroy`

## 5. 写回契约（必须理解）

### 5.1 成功语义

执行类接口成功时（返回 `0` 且结果状态为成功）：

- 几何数组有效数据写回至前缀
- `QemMeshView.num_vertices` / `QemMeshView.num_indices` 回写为输出规模
- 场景执行逐 mesh 产出 `QemSceneMeshResult`（含 `source/requested/output/status`）

### 5.2 失败语义

- 计数字段保持调用前值
- 缓冲区可能存在部分中间写入
- 场景总状态为首个失败码，但其他 mesh 可能已成功

**建议**：对输入几何做快照，按 mesh 粒度回滚，而非仅看全局状态。

---

## 6. 容量契约与两段式调用

所有 `out_xxx + capacity + out_xxx_count` 风格接口建议按两段式使用：

1. 探测数量（空缓冲区或 0 容量）
2. 分配后再次调用

`qem_scene_export_statistics_csv` 同理：

1. `out_buffer = nullptr` 获取 `out_required_size`
2. 重新分配并写入 CSV

容量不足返回 `QEM_STATUS_INSUFFICIENT_BUFFER`。

---

## 7. 并行与执行选项

`QemSceneExecutionOptions` 建议按以下原则配置：

- `enable_parallel`：默认可开启
- `max_parallel_tasks`：不超过业务线程池上限
- `retry_count`：用于短暂失败重试
- `fallback_relaxation_step`：用于降级放宽执行条件

若希望结果更可复现，建议固定执行参数与输入顺序。

---

## 8. 错误处理规范

统一状态码：

- `QEM_STATUS_SUCCESS = 0`
- `QEM_STATUS_INVALID_ARGUMENT = -1`
- `QEM_STATUS_PANIC = -2`
- `QEM_STATUS_INSUFFICIENT_BUFFER = -3`

处理建议：

1. 同时检查函数返回值与输出结构状态
2. 失败时记录 `QemSceneMeshResult[]` 与关键入参
3. 必要时调用 `qem_get_last_result` 补充上下文信息

---

## 9. 从旧接口迁移到新标准

若旧代码仍存在场景树调用，请按下表迁移：

- `QemSceneView` → `QemSceneGraphView`
- `qem_scene_extract_features` → `qem_scene_graph_extract_features`
- `qem_scene_compute_decisions` → `qem_scene_graph_compute_decisions`
- `qem_scene_apply_decisions(_ex)` → `qem_scene_graph_apply_decisions(_ex)`
- `qem_scene_simplify(_ex)` → `qem_scene_graph_simplify(_ex)`

迁移后应删除旧符号加载逻辑，避免运行时分支歧义。
