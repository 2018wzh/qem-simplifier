# qem-simplifier 跨引擎接入指南

> 目标：通过 C ABI 在任意引擎中接入 `qem-simplifier`。

## 1. 接入前提

- 已有三角网格数据（顶点 + 三角索引）。
- 可以调用 C 接口（`extern "C"` / P/Invoke / FFI）。
- 能加载动态库（Windows `.dll` / Linux `.so` / macOS `.dylib`）。

## 2. 数据结构

`qem_simplify` 会**原地修改**输入数组，并将有效数据压缩到数组前缀。

### 2.0 基础格式约定（必须遵守）

- 所有计数字段均为**元素个数**，不是字节数。
- 标量类型宽度固定：
  - `float` = 32-bit IEEE 754（`f32`）
  - `uint32_t` = 32-bit 无符号整数
  - `int32_t` = 32-bit 有符号整数
- 输入数组必须为**连续内存**：
  - `vertices`、`indices`、`material_ids` 在执行期间可写；
  - `attribute_weights` 在执行期间可读。
- 调用期间所有指针必须保持有效，不得被移动、释放或复用。
- 建议输入中不要包含 `NaN/Inf`，避免不可预期的简化结果。

### 2.1 网格输入 `QemMeshView`

| 字段 | 类型 | 是否必填 | 约束 |
|---|---|---|---|
| `vertices` | `float*` | 是 | 长度必须为 `num_vertices * (3 + num_attributes)` |
| `num_vertices` | `uint32_t` | 是 | 必须 `> 0` |
| `indices` | `uint32_t*` | 是 | 长度必须为 `num_indices` |
| `num_indices` | `uint32_t` | 是 | 必须 `> 0` 且 `num_indices % 3 == 0` |
| `material_ids` | `int32_t*` | 是 | 长度必须为 `num_indices / 3` |
| `num_attributes` | `uint32_t` | 是 | 每顶点附加属性总维度（不是通道数） |
| `attribute_weights` | `float*` | 条件必填 | 长度必须为 `num_attributes`；当 `num_attributes > 0` 时不可为 `nullptr` |

#### 2.1.1 顶点流布局公式（`vertices`）

- 顶点步长（以 `float` 为单位）：`stride = 3 + num_attributes`
- 第 `i` 个顶点位置：
  - `x = vertices[i * stride + 0]`
  - `y = vertices[i * stride + 1]`
  - `z = vertices[i * stride + 2]`
- 第 `i` 个顶点第 `k` 个附加属性（`k` 从 0 开始）：
  - `attr[k] = vertices[i * stride + 3 + k]`

示例（`num_attributes = 5`，属性定义为 Normal(3) + UV0(2)）：

- 单顶点布局：`[px, py, pz, nx, ny, nz, u0, v0]`
- `stride = 8`

#### 2.1.2 索引与材质映射（`indices` + `material_ids`）

- 三角形数量：`num_triangles = num_indices / 3`
- 第 `t` 个三角形：
  - 顶点索引：`indices[3*t + 0]`, `indices[3*t + 1]`, `indices[3*t + 2]`
  - 材质 ID：`material_ids[t]`
- 每个索引都应满足：`indices[i] < num_vertices`。

#### 2.1.3 属性权重映射（`attribute_weights`）

- `attribute_weights` 与附加属性维度一一对应，长度严格等于 `num_attributes`。
- 权重按 `vertices` 中附加属性顺序对齐。
- 若你以“通道”管理属性（如 Normal、UV、Color），请将通道权重展开到每个标量维度。

附加属性本质上按“**标量维度向量**”参与误差计算，不区分语义类型；
因此是否“重要”完全由数值尺度和 `attribute_weights` 共同决定。

示例：

- 属性顺序：Normal(3) + UV0(2) + Color(4)，则 `num_attributes = 9`
- 可选权重：`[1,1,1, 0.5,0.5, 0.2,0.2,0.2,0.2]`

#### 2.1.4 附加属性格式细化（语义与打包）

##### A) 语义类型与建模建议

- **连续属性（建议直接参与优化）**：
  - 法线（Normal）、切线（Tangent）、UV、顶点色、AO/曲率等连续值。
- **离散属性（建议谨慎处理）**：
  - 材质槽索引、骨骼 ID、分组标签等离散值。
  - 这类属性更推荐放在 `material_ids` 或外部映射中，而不是直接拼进顶点浮点属性。

##### B) 常见属性通道模板（推荐）

| 语义 | 维度 | 推荐值域/约定 | 打包示例 | 权重建议（起步） |
|---|---:|---|---|---|
| Normal | 3 | `[-1, 1]`，尽量单位化 | `nx, ny, nz` | `0.5 ~ 2.0` |
| Tangent(+Sign) | 4 | xyz 在 `[-1,1]`，w 为 `±1` | `tx, ty, tz, tw` | `0.25 ~ 1.0` |
| UV0/UV1 | 2/2 | 贴图坐标，建议统一量纲 | `u0,v0,u1,v1` | `0.1 ~ 1.0` |
| Color | 3/4 | `[0,1]` 线性空间 | `r,g,b[,a]` | `0.05 ~ 0.5` |
| AO/Mask | 1 | `[0,1]` | `ao` | `0.05 ~ 0.3` |

> 若某通道原始数据是压缩格式（如 8-bit 颜色、10-10-10 法线），请先解码到 `f32` 再拼接。

##### C) 通道拼接与维度计算规则

- `num_attributes` 必须等于所有附加通道维度之和：
  - 例如：Normal(3) + UV0(2) + UV1(2) + Color(4) => `num_attributes = 11`
- 顶点记录中必须使用**固定顺序**，且全网格一致。
- `attribute_weights` 必须与该顺序逐维严格对齐。

##### D) 缝（Seam）与硬边规则（重要）

- `qem_simplifier` 的附加属性是**按顶点存储**，不是按角点（corner）存储。
- 若同一位置在不同三角形上属性不同（如 UV seam、硬边法线）：
  - 需要在输入网格中**拆分顶点**（位置可相同，属性不同）。
- 否则会出现 seam 被“平均化”或属性串扰。

##### E) 权重为 0 的特殊行为

- 当前实现中，只要任一属性维度权重为 `0`，会进入“零权重路径”。
- 这会改变属性更新策略（更偏向拷贝已有属性值而非连续优化）。
- 实务建议：
  - 连续属性优先使用小正数（如 `1e-3`）而不是 `0`；
  - 确需离散属性时，优先拆到外部结构或分批处理。

#### 2.1.5 最小输入校验清单（建议在调用前执行）

- `vertices/indices/material_ids` 非空。
- `num_vertices > 0`。
- `num_indices > 0 && num_indices % 3 == 0`。
- `material_ids` 长度为 `num_indices / 3`。
- `vertices` 长度为 `num_vertices * (3 + num_attributes)`。
- `num_attributes > 0` 时，`attribute_weights` 非空且长度为 `num_attributes`。
- `indices` 全部落在 `[0, num_vertices)`。

### 2.2 简化参数 `QemSimplifyOptions`

- `target_vertices`: 目标顶点数（0 表示不以此项作为硬目标）。
- `target_triangles`: 目标三角形数（0 表示不以此项作为硬目标）。
- `target_error`: 目标误差阈值。
- `min_vertices`: 最小保留顶点数（下限保护）。
- `min_triangles`: 最小保留三角形数（下限保护）。
- `limit_error`: 误差上限，超过则停止。
- `edge_weight`: 边界/边特征权重。
- `max_edge_length_factor`: 最大边长因子（0 表示禁用）。
- `preserve_surface_area`: 是否启用表面积保护（0/1）。

补充约束与建议：

- `target_vertices`、`target_triangles`、`min_vertices`、`min_triangles` 都是**数量**而非比例。
- `target_vertices = 0` 且 `target_triangles = 0` 时，通常应至少提供 `target_error` 或 `limit_error` 作为停止条件。
- `preserve_surface_area`：`0 = false`，非 `0 = true`。
- 建议保持参数尺度一致（例如网格坐标单位统一为米或厘米），便于稳定控制误差阈值。

### 2.3 输出 `QemSimplifyResult`

- `status`: 本次调用状态码。
- `max_error`: 实际最大误差。
- `num_vertices`: 输出有效顶点数。
- `num_indices`: 输出有效索引数。
- `num_triangles`: 输出有效三角形数。

调用后请只读取前缀：

- 顶点有效区间：`vertices[0 .. num_vertices * (3 + num_attributes))`
- 索引有效区间：`indices[0 .. num_indices)`
- 材质有效区间：`material_ids[0 .. num_triangles)`

补充说明：

- 输出满足：`num_indices == num_triangles * 3`。
- 数组后缀数据可能是旧值或无效值，必须忽略。
- 简化后索引与顶点顺序可能重排，属于正常行为。

## 3. 调用流程（通用于任意引擎）

1. 加载动态库并绑定符号。  
2. 可选：`register_log_callback` 注册日志回调。  
3. `qem_context_create` 创建上下文。  
4. 组装 `QemMeshView` 与 `QemSimplifyOptions`。  
5. 调用 `qem_simplify`。  
6. 从 `QemSimplifyResult` 读取有效数量并回写引擎网格。  
7. `qem_context_destroy` 释放上下文。

## 4. 状态码与错误处理

- `QEM_STATUS_SUCCESS = 0`：成功。
- `QEM_STATUS_INVALID_ARGUMENT = -1`：参数非法（空指针、长度不匹配、索引不是 3 的倍数等）。
- `QEM_STATUS_PANIC = -2`：内部异常被捕获。

建议：

- 同时检查函数返回值和 `out_result.status`。
- 失败时调用 `qem_get_last_result` 获取上下文中的最后结果快照。

## 5. 引擎适配清单

将你的引擎网格适配到以下统一输入即可：

- 顶点坐标转为 `f32`。
- 索引转为 `u32` 且按三角形展开。
- 每三角形提供一个 `material_id`（无材质可统一填 0）。
- 顶点属性通道按固定顺序拼接（例如：法线 3 + UV 2）。
- 为每个属性通道配置权重（`attribute_weights`）。

> 只要满足上述数据契约，渲染引擎类型本身（Unity/Godot/自研）并不影响接入方式。

## 6. 线程与生命周期建议

- 上下文对象（`qem_context_create`）建议按任务/线程独立创建与销毁。
- 输入数组在 `qem_simplify` 执行期间必须保持有效且可写。
- 日志回调是全局注册，建议在进程初始化阶段设置一次。
