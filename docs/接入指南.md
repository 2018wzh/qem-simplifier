# qem-simplifier 跨引擎接入指南

> 目标：通过 C ABI 在任意引擎中接入 `qem-simplifier`。

## 1. 接入前提

- 已有三角网格数据（顶点 + 三角索引）。
- 可以调用 C 接口（`extern "C"` / P/Invoke / FFI）。
- 能加载动态库（Windows `.dll` / Linux `.so` / macOS `.dylib`）。

## 2. 数据结构

`qem_simplify` 会**原地修改**输入数组，并将有效数据压缩到数组前缀。

### 2.1 网格输入 `QemMeshView`

- `vertices`: `float*`
  - 布局：每个顶点连续存储 `3 + num_attributes` 个浮点。
  - 前 3 个为位置 `(x, y, z)`。
  - 后续 `num_attributes` 为自定义属性（如法线、UV、颜色等）。
- `num_vertices`: 顶点数量。
- `indices`: `uint32_t*`
  - 三角面索引，长度为 `num_indices`，必须满足 `num_indices % 3 == 0`。
- `num_indices`: 索引总数。
- `material_ids`: `int32_t*`
  - 长度必须为 `num_indices / 3`（每个三角形一个材质 ID）。
- `num_attributes`: 每顶点附加属性维度。
- `attribute_weights`: `float*`
  - 长度必须为 `num_attributes`。
  - 当 `num_attributes > 0` 时不可为 `nullptr`。

### 2.2 简化参数 `QemSimplifyOptions`

- `target_vertices`: 目标顶点数（0 表示不以此项作为硬目标）。
- `target_triangles`: 目标三角形数（0 表示不以此项作为硬目标）。
- `target_error`: 目标误差阈值。
- `min_vertices`: 最小保留顶点数（下限保护）。
- `min_triangles`: 最小保留三角形数（下限保护）。
- `limit_error`: 误差上限，超过则停止。
- `edge_weight`: 边界/边特征权重。
- `max_edge_length_factor`: 最大边长因子（0 表示禁用）。
- `preserve_surface_area`: 是否启用表面积保护（0/1）。

### 2.3 输出 `QemSimplifyResult`

- `status`: 本次调用状态码。
- `max_error`: 实际最大误差。
- `num_vertices`: 输出有效顶点数。
- `num_indices`: 输出有效索引数。
- `num_triangles`: 输出有效三角形数。

调用后请只读取前缀：

- 顶点有效区间：`vertices[0 .. num_vertices * (3 + num_attributes))`
- 索引有效区间：`indices[0 .. num_indices)`
- 材质有效区间：`material_ids[0 .. num_triangles)`

## 3. 调用流程（通用于任意引擎）

1. 加载动态库并绑定符号。  
2. 可选：`register_log_callback` 注册日志回调。  
3. `qem_context_create` 创建上下文。  
4. 组装 `QemMeshView` 与 `QemSimplifyOptions`。  
5. 调用 `qem_simplify`。  
6. 从 `QemSimplifyResult` 读取有效数量并回写引擎网格。  
7. `qem_context_destroy` 释放上下文。

## 4. 状态码与错误处理

- `QEM_STATUS_SUCCESS = 0`：成功。
- `QEM_STATUS_INVALID_ARGUMENT = -1`：参数非法（空指针、长度不匹配、索引不是 3 的倍数等）。
- `QEM_STATUS_PANIC = -2`：内部异常被捕获。

建议：

- 同时检查函数返回值和 `out_result.status`。
- 失败时调用 `qem_get_last_result` 获取上下文中的最后结果快照。

## 5. 引擎适配清单

将你的引擎网格适配到以下统一输入即可：

- 顶点坐标转为 `f32`。
- 索引转为 `u32` 且按三角形展开。
- 每三角形提供一个 `material_id`（无材质可统一填 0）。
- 顶点属性通道按固定顺序拼接（例如：法线 3 + UV 2）。
- 为每个属性通道配置权重（`attribute_weights`）。

> 只要满足上述数据契约，渲染引擎类型本身（Unity/Godot/自研）并不影响接入方式。

## 6. 线程与生命周期建议

- 上下文对象（`qem_context_create`）建议按任务/线程独立创建与销毁。
- 输入数组在 `qem_simplify` 执行期间必须保持有效且可写。
- 日志回调是全局注册，建议在进程初始化阶段设置一次。
