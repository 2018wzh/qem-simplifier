# qem-simplifier 跨引擎接入指南（写回契约版）

> 本文档用于统一“调用方式 + 写回行为 + 回滚策略”。
> 目标是让引擎侧、CLI 侧、离线流水线在同一契约下稳定接入。

---

## 1. 快速结论（先看这段）

- `qem_simplify` / `qem_scene_apply_decisions(_ex)` / `qem_scene_simplify(_ex)` 会修改几何数组（原地）。
- 从当前版本开始，**成功时会回写 mesh 计数字段**：
  - `QemMeshView.num_vertices`
  - `QemMeshView.num_indices`
- 场景路径下，回写目标是 `QemSceneView.meshes[i].mesh` 或 `QemSceneGraphView.meshes[i].mesh`（逐 mesh 原地更新）。
- 如果失败，`QemMeshView` 计数字段保持调用前值；但数组内容可能已被部分写入，生产环境建议调用方自行做输入快照/事务回滚。

---

## 2. 基础类型与输入前置条件

### 2.1 基础类型

- `float`：32-bit IEEE754
- `uint32_t` / `int32_t`：32-bit
- 所有长度字段单位均为“元素个数”（不是字节）

### 2.2 `QemMeshView` 数组布局

- 顶点步长：`stride = 3 + num_attributes`
- `vertices` 长度必须为：`num_vertices * stride`
- `indices` 长度必须为：`num_indices`，且 `num_indices % 3 == 0`
- `material_ids` 长度必须为：`num_indices / 3`
- 当 `num_attributes > 0` 时，`attribute_weights` 长度必须为 `num_attributes`

### 2.3 调用前最小校验（建议）

- 指针非空：`vertices/indices/material_ids`
- `num_vertices > 0`
- `num_indices > 0 && num_indices % 3 == 0`
- `indices[i] < num_vertices`
- 若有属性：`attribute_weights != nullptr`

---

## 3. 写回契约（核心）

## 3.1 单网格：`qem_simplify`

### 输入

- `mesh` 指向可写的 `QemMeshView`
- `options` 为本次简化目标

### 成功（函数返回 0 且 `out_result.status == 0`）

库保证：

1. 几何数组原地写回，且有效数据位于前缀：
   - 顶点有效区：`vertices[0 .. out_result.num_vertices * (3 + num_attributes))`
   - 索引有效区：`indices[0 .. out_result.num_indices)`
   - 材质有效区：`material_ids[0 .. out_result.num_triangles)`
2. `mesh->num_vertices` 被更新为 `out_result.num_vertices`
3. `mesh->num_indices` 被更新为 `out_result.num_indices`
4. 保证 `out_result.num_indices == out_result.num_triangles * 3`

### 失败（返回负值或 `out_result.status < 0`）

- `mesh->num_vertices` / `mesh->num_indices` 保持调用前值。
- `out_result` 反映失败状态。
- **数组内容不保证可直接继续使用**（可能存在部分中间写入）。

> 建议：在生产批处理中，对输入数组做快照，失败时回滚。

---

## 3.2 场景执行：`qem_scene_apply_decisions(_ex)` / `qem_scene_simplify(_ex)` / `qem_scene_graph_*`

### 成功路径下（按 mesh）

对每个 mesh（`scene.meshes[i].mesh` 或 `scene_graph.meshes[i].mesh`）：

1. 原地写回该 mesh 的数组前缀。
2. 回写：
   - `scene.meshes[i].mesh.num_vertices`
   - `scene.meshes[i].mesh.num_indices`
3. 同步输出 `QemSceneMeshResult[i]`：
   - `source_triangles`（调用前）
   - `requested_triangles`
   - `output_triangles`
   - `status`

### 失败路径下（某个 mesh 失败）

- 该 mesh 的 `num_vertices/num_indices` 保持调用前值。
- `QemSceneMeshResult.status` 标记失败。
- 场景汇总状态 `QemSceneSimplifyResult.status` 为首个失败码。
- 其他 mesh 仍可能已成功执行（按结果逐 mesh 判定）。

> 场景执行是“逐 mesh 独立任务模型”，调用方应按 `QemSceneMeshResult[]` 做细粒度回写/回滚策略，而不是只看总状态。

---

## 3.3 只读接口（不写几何）

以下接口不会修改几何数组与 mesh 计数字段：

- `qem_scene_extract_features`
- `qem_scene_compute_decisions`
- `qem_scene_graph_extract_features`
- `qem_scene_graph_compute_decisions`
- `qem_scene_compute_statistics`
- `qem_scene_export_statistics_csv`

---

## 4. 推荐接入流程

## 4.1 在线/引擎实时路径（单网格）

1. 创建 context：`qem_context_create`
2. 配置 `QemMeshView + QemSimplifyOptions`
3. 调用 `qem_simplify`
4. 若成功：按 `mesh.num_vertices/mesh.num_indices` 或 `out_result` 前缀读回
5. 销毁 context：`qem_context_destroy`

## 4.2 场景离线路径（推荐：通用场景图）

1. 组装 `QemSceneGraphView`（`meshes + nodes + mesh_bindings`）
2. `qem_scene_graph_extract_features`
3. `qem_scene_graph_compute_decisions`
4. （可选）人工审阅/业务重写决策
5. `qem_scene_graph_apply_decisions_ex`
6. `qem_scene_compute_statistics`
7. `qem_scene_export_statistics_csv`

## 4.3 Legacy 场景树路径（兼容保留）

1. `qem_scene_extract_features`
2. `qem_scene_compute_decisions`
3. （可选）人工审阅/业务重写决策
4. `qem_scene_apply_decisions_ex`
5. `qem_scene_compute_statistics`
6. `qem_scene_export_statistics_csv`

---

## 4.4 场景图结构约束（新增）

- `QemSceneGraphView.meshes`：可写 mesh 池（执行阶段会原地写回）。
- `QemSceneGraphView.nodes`：节点层级（`parent_index` + `local_matrix`）。
- `QemSceneGraphView.mesh_bindings`：mesh 绑定关系（`node_index` → `mesh_index`）。
- `mesh_bindings` 可表达“同一 mesh 被多个节点实例化”，用于引擎实例化与 FBX 复用语义。
- 当 `policy.use_world_scale != 0` 时，权重会结合节点累计变换与可选 `mesh_to_node_matrix`。

---

## 5. 容量契约与两段式调用

- 带 `out_xxx + capacity + out_xxx_count` 的接口支持“先探测数量，再分配缓冲区”。
- `qem_scene_export_statistics_csv` 支持：
  1) `out_buffer = nullptr` 获取 `out_required_size`
  2) 分配后再次调用写入 CSV

当容量不足时，返回 `QEM_STATUS_INSUFFICIENT_BUFFER`。

---

## 6. 状态码与错误处理

- `QEM_STATUS_SUCCESS = 0`
- `QEM_STATUS_INVALID_ARGUMENT = -1`
- `QEM_STATUS_PANIC = -2`
- `QEM_STATUS_INSUFFICIENT_BUFFER = -3`

建议：

- 同时检查“函数返回值 + out_result.status”。
- 上下文级故障排查可调用 `qem_get_last_result`。
- 批处理场景请记录每 mesh 的 `QemSceneMeshResult`。

---

## 7. 线程与回调

- `qem_context_set_progress_callback` 绑定在 context 上，适合多任务隔离。
- 回调在线程内触发；UI 层请自行跨线程派发。
- 日志回调是全局注册，建议在进程初始化时注册一次。

---

## 8. 版本与兼容性

- 当前 ABI 版本：`6`
- 建议启动时先调用 `qem_get_abi_version()` 校验后再启用能力。
