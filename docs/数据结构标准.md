# qem-simplifier 数据结构标准（新 API 标准 / ABI v7）

> 本文档定义 C ABI 层的数据结构与字段语义，并统一内存契约、写回语义与兼容策略。
>
> 目标：为引擎插件、离线流水线和工具链接入提供一致、可审计、可升级的结构标准。

---

## 1. ABI 与兼容策略

- 当前 ABI 版本：`7`（`qem_get_abi_version()`）。
- 所有跨语言结构体使用 `#[repr(C)]`（C 布局）。
- 兼容原则：
  - **新增字段**：只能通过新增结构体或新增函数实现，不在已发布结构体中插入字段。
  - **变更字段语义**：必须提升 ABI 大版本。
  - **删除字段/函数**：必须提升 ABI 大版本。

---

## 2. 基础标量与内存

- `float`：32-bit IEEE 754。
- `uint32_t` / `int32_t`：32-bit。
- `uint64_t`：64-bit。
- 指针字段均表示调用方管理的外部内存，库不负责分配/释放调用方输入数组。
- 除明确说明外，长度字段单位均为“元素个数”，不是字节数。

---

## 3. 回调类型标准

### 3.1 日志回调

- `LogCallback = void(*)(const char*)`
- 仅用于输出文本日志。

### 3.2 进度回调

- `ProgressCallback = void(*)(const QemProgressEvent*, void*)`
- 通过上下文绑定：
  - `qem_context_set_progress_callback(context, callback, user_data)`
  - `qem_context_clear_progress_callback(context)`

### QemProgressEvent 字段标准

| 字段 | 类型 | 说明 |
|---|---|---|
| `scope` | `uint32_t` | 范围：`QEM_PROGRESS_SCOPE_MESH` 或 `QEM_PROGRESS_SCOPE_SCENE` |
| `stage` | `uint32_t` | 阶段：`BEGIN / UPDATE / END` |
| `percent` | `float` | 进度百分比，规范化到 `[0,1]` |
| `mesh_index` | `uint32_t` | 当前处理 mesh 索引（场景模式） |
| `mesh_count` | `uint32_t` | 总 mesh 数（场景模式） |
| `source_triangles` | `uint32_t` | 输入三角数 |
| `target_triangles` | `uint32_t` | 目标三角数 |
| `output_triangles` | `uint32_t` | 当前/最终输出三角数 |
| `status` | `int32_t` | 当前阶段状态码 |

> 线程模型：回调在算法执行线程触发；调用方负责线程安全与跨线程派发。

## 4. 单网格结构标准

### 4.1 QemMeshView

| 字段 | 类型 | 约束 |
|---|---|---|
| `vertices` | `float*` | 长度 = `num_vertices * (3 + num_attributes)` |
| `num_vertices` | `uint32_t` | `> 0` |
| `indices` | `uint32_t*` | 长度 = `num_indices`，且 `num_indices % 3 == 0` |
| `num_indices` | `uint32_t` | `> 0` |
| `material_ids` | `int32_t*` | 长度 = `num_indices / 3` |
| `num_attributes` | `uint32_t` | 每顶点附加属性维度 |
| `attribute_weights` | `const float*` | 当 `num_attributes > 0` 时长度必须为 `num_attributes` |

### 4.2 QemSimplifyOptions

| 字段 | 类型 | 说明 |
|---|---|---|
| `target_vertices` | `uint32_t` | 目标顶点数（0 表示不作为硬目标） |
| `target_triangles` | `uint32_t` | 目标三角数（0 表示不作为硬目标） |
| `target_error` | `float` | 目标误差阈值 |
| `min_vertices` | `uint32_t` | 顶点下限 |
| `min_triangles` | `uint32_t` | 三角下限 |
| `limit_error` | `float` | 误差硬上限 |
| `edge_weight` | `float` | 边约束权重 |
| `max_edge_length_factor` | `float` | 最大边长因子 |
| `preserve_surface_area` | `uint8_t` | `0/1` |

### 4.3 QemSimplifyResult

| 字段 | 类型 | 说明 |
|---|---|---|
| `status` | `int32_t` | 状态码 |
| `max_error` | `float` | 本次简化误差 |
| `num_vertices` | `uint32_t` | 输出有效顶点数 |
| `num_indices` | `uint32_t` | 输出有效索引数 |
| `num_triangles` | `uint32_t` | 输出有效三角数 |

## 5. 场景图结构标准

### 5.1 QemSceneMeshView

- `mesh_id`：调用方定义的稳定 ID（建议用于资产映射/回写）。
- `mesh`：对应的 `QemMeshView`。

### 5.2 QemSceneGraphNodeView

| 字段 | 类型 | 说明 |
|---|---|---|
| `parent_index` | `int32_t` | 父节点索引；根节点可设 `-1` |
| `local_matrix` | `float[16]` | 节点局部变换矩阵 |

### 5.3 QemSceneGraphMeshBindingView

| 字段 | 类型 | 说明 |
|---|---|---|
| `node_index` | `uint32_t` | 绑定目标节点索引 |
| `mesh_index` | `uint32_t` | 绑定的 mesh 索引（指向 `meshes`） |
| `mesh_to_node_matrix` | `float[16]` | 可选 mesh→node 附加变换 |
| `use_mesh_to_node_matrix` | `uint8_t` | `0/1`，是否启用附加变换 |

### 5.4 QemSceneGraphView

| 字段 | 类型 | 说明 |
|---|---|---|
| `meshes` | `QemSceneMeshView*` | mesh 池（执行阶段原地写回） |
| `num_meshes` | `uint32_t` | mesh 数量，必须 `> 0` |
| `nodes` | `const QemSceneGraphNodeView*` | 节点数组（可空，仅当 `num_nodes == 0`） |
| `num_nodes` | `uint32_t` | 节点数量 |
| `mesh_bindings` | `const QemSceneGraphMeshBindingView*` | 绑定数组（可空，仅当 `num_mesh_bindings == 0`） |
| `num_mesh_bindings` | `uint32_t` | 绑定数量 |

> 设计意图：将“节点层级”与“mesh 绑定”解耦，天然支持实例复用（同一 mesh 多节点绑定），更贴合游戏引擎与 FBX 语义。

### 5.5 QemScenePolicy

| 字段 | 类型 | 说明 |
|---|---|---|
| `target_triangle_ratio` | `float` | 场景总体目标比例，建议 `[0,1]` |
| `target_total_triangles` | `uint64_t` | 场景目标总面数（>0 时优先） |
| `min_mesh_ratio` | `float` | 单 mesh 下界比例 |
| `max_mesh_ratio` | `float` | 单 mesh 上界比例 |
| `min_triangles_per_mesh` | `uint32_t` | 单 mesh 绝对最小保留面数 |
| `weight_mode` | `uint32_t` | 权重模式常量 |
| `weight_exponent` | `float` | 权重指数（非线性放大/抑制） |
| `use_world_scale` | `uint8_t` | 是否启用节点 world scale 影响 |
| `enable_parallel` | `uint8_t` | 执行阶段是否并行 |
| `max_parallel_tasks` | `uint32_t` | 执行线程上限（0=默认线程池） |
| `external_importance_weights` | `const float*` | `QEM_SCENE_WEIGHT_EXTERNAL` 模式下的权重数组 |
| `external_importance_count` | `uint32_t` | 外部权重数组长度，需 ≥ `num_meshes` |

### 5.6 QemSceneExecutionOptions

执行阶段增强参数：

| 字段 | 类型 | 说明 |
|---|---|---|
| `enable_parallel` | `uint8_t` | 执行层并行开关 |
| `max_parallel_tasks` | `uint32_t` | 线程上限（0=默认） |
| `retry_count` | `uint32_t` | 单 mesh 失败重试次数（>=1） |
| `fallback_relaxation_step` | `float` | 重试时目标放宽步长 |

### 5.7 QemSceneMeshFeature

场景特征提取阶段输出：

| 字段 | 类型 | 说明 |
|---|---|---|
| `mesh_index` | `uint32_t` | mesh 索引 |
| `mesh_id` | `uint32_t` | 调用方定义 ID |
| `source_triangles` | `uint32_t` | 初始面数 $T_i$ |
| `instance_count` | `uint32_t` | 场景实例数量 |
| `world_scale_sum` | `double` | 变换尺度累计值 |
| `bbox_volume` | `double` | 包围盒体积 $V_i$ |
| `importance_weight` | `double` | 预算分配权重（含指数） |

### 5.8 QemSceneMeshDecision

- 决策层输出，执行层输入。
- 关键字段：`mesh_index`、`keep_ratio`、`target_triangles`。

### 5.9 QemSceneMeshResult / QemSceneSimplifyResult

- `QemSceneMeshResult`：每个 mesh 的执行结果。
- `QemSceneSimplifyResult`：场景级聚合统计（总输入/目标/输出三角数等）。

### 5.10 QemSceneMeshStatistics

统计阶段输出（每 mesh 一条）：

| 字段 | 类型 | 说明 |
|---|---|---|
| `mesh_index` | `uint32_t` | mesh 索引 |
| `mesh_id` | `uint32_t` | 调用方定义 ID |
| `status` | `int32_t` | 该 mesh 执行状态 |
| `source_triangles` | `uint32_t` | 输入面数 |
| `target_triangles` | `uint32_t` | 请求目标面数 |
| `output_triangles` | `uint32_t` | 输出面数 |
| `source_effective_triangles` | `double` | 有效输入面数 |
| `target_effective_triangles` | `double` | 有效目标面数 |
| `output_effective_triangles` | `double` | 有效输出面数 |
| `target_ratio` | `float` | 目标保留比例 |
| `achieved_ratio` | `float` | 实际保留比例 |
| `budget_deviation` | `float` | 预算偏差 $(output-target)/target$ |
| `max_error` | `float` | 该 mesh 最大误差 |

### 5.11 QemSceneStatisticsSummary

统计汇总输出：

| 字段 | 类型 | 说明 |
|---|---|---|
| `status` | `int32_t` | 场景统计状态（若存在失败 mesh，为首个失败码） |
| `num_meshes` | `uint32_t` | 统计对象数 |
| `num_failed_meshes` | `uint32_t` | 失败对象数 |
| `num_simplified_meshes` | `uint32_t` | 成功且输出面数下降的对象数 |
| `source_triangles` | `uint64_t` | 总输入面数 |
| `target_triangles` | `uint64_t` | 总目标面数 |
| `output_triangles` | `uint64_t` | 总输出面数 |
| `target_scene_ratio` | `float` | 目标场景保留比例 |
| `achieved_scene_ratio` | `float` | 实际场景保留比例 |
| `target_hit_ratio` | `float` | 目标达成率（`output/target`） |
| `mean_abs_budget_deviation` | `float` | 平均绝对预算偏差 |
| `max_abs_budget_deviation` | `float` | 最大绝对预算偏差 |
| `mean_max_error` | `float` | 成功对象平均最大误差 |

## 6. 缓冲区容量契约

凡是输出数组参数，均采用：

- `out_xxx`：可空；
- `xxx_capacity`：调用方声明可写容量；
- `out_xxx_count`：库返回实际需求数量。

当容量不足时返回：`QEM_STATUS_INSUFFICIENT_BUFFER`。

统计导出函数同样遵循该契约：

- `qem_scene_compute_statistics`：支持先拿 `out_statistics_count`，再分配 `out_statistics`。
- `qem_scene_export_statistics_csv`：支持先传 `out_buffer=nullptr` 获取 `out_required_size`，再二次调用写入文本。

### 6.1 几何写回契约

### 单网格 `qem_simplify`

- 成功时：
  - 原地写回数组前缀；
  - 回写 `mesh.num_vertices`、`mesh.num_indices`；
  - 并满足 `num_indices == num_triangles * 3`。
- 失败时：
  - `mesh.num_vertices`、`mesh.num_indices` 保持调用前值；
  - 数组内容不保证可直接继续使用（建议调用方做快照并在失败时回滚）。

### 场景执行 `qem_scene_graph_apply_decisions(_ex)` / `qem_scene_graph_simplify(_ex)`

- 按 mesh 独立执行并写回 `scene_graph.meshes[i].mesh`。
- 对成功 mesh：回写 `num_vertices/num_indices`；失败 mesh 保持调用前计数。
- 调用方应以 `QemSceneMeshResult.status` 作为逐 mesh 是否采用写回结果的依据。

## 7. 状态码标准

- `QEM_STATUS_SUCCESS = 0`
- `QEM_STATUS_INVALID_ARGUMENT = -1`
- `QEM_STATUS_PANIC = -2`
- `QEM_STATUS_INSUFFICIENT_BUFFER = -3`

---

## 8. 建议的接入校验流程

1. 加载库后先检查 ABI 版本。
2. 在调用前进行结构体字段与数组长度校验。
3. 首次集成建议开启日志回调与进度回调。
4. 对失败调用，记录输入参数快照与状态码。
5. 对场景流程，优先两阶段：先 `compute_decisions` 再 `apply_decisions`（scene_graph）。

---

## 9. 管线级分步并行建议

推荐流程（场景图）：

1. `qem_scene_graph_extract_features`；
2. `qem_scene_graph_compute_decisions`；
3. `qem_scene_graph_apply_decisions` / `qem_scene_graph_apply_decisions_ex`。

该流程在工程上具备：低内存占用、多核扩展性强、对象间无拓扑缝合风险。
